## Сообщить об уязвимости

**Пожалуйста, не создавайте публичные GitHub Issues для уязвимостей безопасности.**

### Как сообщить

1. Используйте **[GitHub Private Security Advisory]**:
   `Security → Advisories → New draft security advisory`
   в репозитории
   [Weather-Dashboard-Open-Meteo-API](https://github.com/Loliconus/Weather-Dashboard-Open-Meteo-API/security/advisories/new)

2. Или напишите напрямую мейнтейнеру через GitHub ([@Loliconus](https://github.com/Loliconus)).

### Что включить в отчёт

- Тип уязвимости (XSS, supply chain, утечка данных и т.п.)
- Путь к уязвимому коду (файл, строка)
- Шаги воспроизведения
- Возможное влияние
- Предлагаемое исправление (если есть)

### Ожидаемые сроки

| Событие                          | Срок     |
|----------------------------------|----------|
| Подтверждение получения отчёта   | 48 часов |
| Оценка и присвоение серьёзности  | 7 дней   |
| Исправление (критические)        | 14 дней  |
| Исправление (некритические)      | 30 дней  |
| Публичное раскрытие              | После патча |

## Модель угроз

Этот проект — **статический сайт** на GitHub Pages. Основные поверхности атаки:

| Поверхность | Меры защиты |
|-------------|-------------|
| Supply chain (зависимости Python) | `uv.lock`, SHA-пиннинг Actions |
| Supply chain (JS CDN)             | SRI `integrity` для Chart.js |
| GitHub Actions                    | SHA-пиннинг всех `uses:`, OIDC (нет секретов) |
| API ключи                         | Open-Meteo не требует ключей |
| XSS                               | Jinja2 `autoescape=True`, Content-Security-Policy |
| Данные пользователя               | Не собираются; `localStorage` только на устройстве |

## Политика раскрытия

Мы следуем принципу **Coordinated Vulnerability Disclosure (CVD)**:
уязвимость публично раскрывается после выпуска патча
или через 90 дней с момента получения отчёта (что наступит раньше).
